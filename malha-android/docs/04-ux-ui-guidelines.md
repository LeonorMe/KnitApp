# UX/UI Guidelines

## Design Goal

The app should feel like a calm place to continue a handmade project. It should reduce confusion, not create a productivity dashboard that makes crafting feel like work.

## Navigation

Use bottom navigation with 3-5 destinations:

- Home
- Projects
- Patterns
- Materials
- Settings

Community can be added later as a fifth tab or as part of Patterns.

## Project Execution Screen

This is the most important screen.

Required elements:

- Project name.
- Current row or step number.
- Large instruction text.
- Progress indicator.
- Previous and next controls.
- Mark done button.
- Notes button or expandable notes area.

Design rules:

- Make step text highly readable.
- Keep controls fixed and easy to reach.
- Save progress immediately.
- Resume to the exact last step.

## Visual Style

Brand direction:

- Warm, soft, textile-inspired.
- Gender-inclusive.
- Calm rather than childish.
- Craft-related without being cluttered.

Suggested palette:

- Warm cream background.
- Deep sage green for progress and primary actions.
- Muted indigo for assistant/AI elements.
- Soft clay or beige for secondary surfaces.
- Dusty lavender for achievements and highlights.

Avoid:

- Pink-heavy identity.
- Overly decorative craft clutter.
- Harsh gamification colors.
- Urgent red/orange unless there is a real warning.

## Accessibility

- Minimum touch target: 48dp.
- Prefer 56dp for main knitting controls.
- Support system font scaling.
- Avoid thin fonts.
- Use strong text contrast.
- Do not rely only on color to show state.

## Tone

The app should sound:

- Calm.
- Clear.
- Non-judgmental.
- Helpful.

Avoid guilt-based copy. The app can nudge users toward finishing, but it should never shame them for having unfinished projects or unused yarn.

## AI UX

Assistant suggestions should be visually distinct from verified pattern instructions.

Use labels such as:

- "Aidi suggests"
- "AI draft"
- "Needs review"
- "Community verified"

AI output must always be editable or dismissible.
